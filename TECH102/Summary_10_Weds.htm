A DIGITAL PHOTOGRAPH

Consider an image held close for careful examination, e.g. a printed photograph. First, how close should we assume people are holding the photo? Recall we measured approximately our near points (closest point at which an object can be focused) in a previous class. The nearpoint changes with age – elderly people have to hold a book or newspaper at some distance to focus it. Generally 25 cm is used as a benchmark for calculations, but younger people can actually hold a photograph quite close and see more details. Students in Tech 102 mostly had near points closer than 25 cm for example.

EXPECTED FILE SIZES

We estimated that a 200x400, 24-bit colour image would occupy

200x400x3 bytes = 240,000 bytes.

Using our RGB simulator we created a uniform green image and saved is as a bitmap. The file size was 244 KB. Seems like a lot for a small patch of colour!

 

MOVIES

Note that the human eye is actually a movie camera. Experiments show that the eye snaps about 20 frames per second. Thus changes from one frame to the next at more than 20 frames per second give the perception of continuous motion. Modern movies are displayed at 24 frames per second.

Exercise: Calculate the number of bytes in a two hour movie displayed at XGA resolution at 24 frames per second.

3 bytes/pixel x 1024 x 768 pixels x 24 frames/sec x 3600 sec/hr x 2 hr = 408 GB!!!

 

COMPRESSION
